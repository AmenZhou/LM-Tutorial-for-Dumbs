Phase 3 — Powerful Models
When straight lines aren't enough. Trees, forests, boosting, clustering, and more — each one a different strategy for finding patterns. CloudSync needs to detect incidents, segment users, and filter spam — problems that linear models can't solve.
Planned · 8 weeks
W1
🎄 "The 20 Questions Incident Triage Game" — Decision Trees
Build a tree by choosing splits · gini impurity · information gain · depth vs overfitting · visualise each branch
Real example: a tree that diagnoses CloudSync outages — CPU spike? DB timeout? Memory leak? Each split narrows it down
Planned
W2
🗳️ "Committee Vote" — Random Forests
100 weak trees vote · bagging · feature randomness · out-of-bag error · why the crowd beats any single expert
Real example: 100 incident detectors each see a random subset of features — their majority vote is more reliable than any single one
Planned
W3
📈 "Fix Your Mistakes" — Gradient Boosting / XGBoost
Start with a bad model · add model 2 that learns model 1's errors · add model 3 · watch accuracy compound
Real example: linear model predicts churn badly. XGBoost layers 100 corrections on top of it and wins the leaderboard
Planned
W4
👥 "Find Your Tribe" — K-Means Clustering
Drag centroids · watch clusters form · elbow method · unsupervised learning — no labels needed
Real example: segment 50,000 CloudSync users into usage personas without being told how many groups exist
Planned
W5
✏️ "Draw the Perfect Boundary" — Support Vector Machines
Drag the decision boundary · maximise the margin · support vectors highlighted · kernel trick for curved boundaries
Real example: separate abusive API callers from legitimate users — the boundary must be as far from both groups as possible
Planned
W6
🔎 "The Probability Detective" — Naive Bayes
Conditional probability · Bayes theorem with emoji dice · word frequency classifier · why "naive" still works
Real example: classify CloudSync support tickets as billing / technical / feature request using word counts
Planned
W7
📍 "How Similar Are These Users?" — K-Nearest Neighbours
Distance metric game · curse of dimensionality · k selection · plot neighbours in 2D space
Real example: a user churned. Find the 5 most similar users — are they showing the same warning signs?
Planned
W8
🏆 Model Showdown Capstone
Same churn dataset · 6 models compete · live accuracy leaderboard · accuracy vs speed vs interpretability trade-off matrix
Real example: which model do you ship to production? The most accurate? The fastest? The most explainable to your CEO?
Planned

Phase 4 — Neural Networks & Deep Learning
The technology behind ChatGPT, image recognition, and self-driving cars. Starts with a single neuron (which you already understand from Phase 1) and builds up to convolutional networks and PyTorch. CloudSync wants to classify dashboard screenshots and predict server load sequences.
Planned · 7 weeks
W1
⚡ "One Neuron" — The Perceptron
A single neuron as a weighted sum + activation · direct connection to Phase 1 matrix math · train on XOR problem
Real example: one neuron that flags high-risk deployments — input: error rate, deploy frequency, team size → output: risk score
Planned
W2
🏗️ "Build It Layer by Layer" — Deep Networks
Add hidden layers one at a time · watch accuracy jump · visualise what each layer learns · vanishing gradient preview
Real example: predicting user lifetime value — 1 layer barely works, 3 layers captures complex usage patterns
Planned
W3
⚡ "Why ReLU Won" — Activation Functions Game
Sigmoid gradient vanishes in deep nets (watch it happen) · ReLU fixes it · compare training speed · gradient flow visualiser
Real example: a 10-layer network with sigmoid barely learns. Swap to ReLU — same architecture, 10× faster convergence
Planned
W4
↩️ "Follow the Error Backwards" — Backpropagation
Step through backprop manually on a 3-node net · chain rule as blame assignment · which weight caused the most damage?
Real example: model predicted 0.9 (very likely to buy), actual was 0 (didn't buy). Trace the error back to see which weights to fix
Planned
W5
👁️ "How Does the Model See?" — Convolutional Networks
Drag filters over an image · edge detection → shape → object builds up · interactive feature map viewer
Real example: classify CloudSync error screenshots as "crashed", "degraded", or "healthy" from a dashboard screenshot
Planned
W6
🔄 "Predict Next Month" — Recurrent Networks / LSTM
Sequential data · hidden state · LSTM gates (forget / input / output) · compare LSTM vs linear on time-series
Real example: CloudSync server load follows weekly cycles. LSTM sees the sequence; linear regression treats each month as independent
Planned
W7
🔥 PyTorch Capstone — Train a Real Neural Network
Browser simulator + Python notebook · MNIST handwriting classifier · compare against Phase 2 logistic regression
Real example: handwriting digit recognition — the classic "hello world" of deep learning, done properly
Planned

Phase 5 — NLP & Language Models
How computers read text — from counting words to understanding meaning to the transformer architecture that powers GPT. CloudSync gets thousands of support tickets and App Store reviews per day. This phase teaches the model to read them.
Planned · 6 weeks
W1
🔤 "Words as Numbers" — Tokenisation & Bag of Words
Turn sentences into vectors · one-hot encoding · vocabulary size trade-off · see why "server down" and "outage" become neighbours
Real example: CloudSync support ticket "DB connection timeout" → a vector of word counts the model can actually learn from
Planned
W2
📊 "Which Words Actually Matter?" — TF-IDF Game
"the" appears 10,000× but means nothing · TF-IDF punishes common words · interactive word weight slider
Real example: distinguish bug reports from feature requests — "crash" and "error" score high; "the" and "and" score near zero
Planned
W3
🌌 "Words in Space" — Word Embeddings
Drag words around 2D embedding space · king − man + woman ≈ queen · semantic similarity visualiser
Real example: "AWS" + "outage" ≈ "incident". The model learns meaning from context, not definitions
Planned
W4
😡 "Is This Review Angry?" — Sentiment Classifier
Train on App Store reviews · positive vs negative · confidence score · misclassification explorer
Real example: CloudSync gets 200 App Store reviews/day. Auto-flag negatives for the support team before they go viral
Planned
W5
👁️ "Pay Attention" — Transformer Architecture Game
Self-attention visualiser · "bank" meaning changes with context · query/key/value intuition · multi-head attention
Real example: "fix the connection" — connection to DB or to the user? The transformer figures it out from surrounding words
Planned
W6
🎛️ Fine-Tuning Capstone — Teach GPT to Answer CloudSync FAQs
Prompt engineering · few-shot vs fine-tuning · evaluate base model vs fine-tuned on real Q&A pairs
Real example: base GPT gives generic answers. Fine-tuned on 200 CloudSync FAQ pairs, it answers product questions accurately
Planned

W3

Accuracy timeline · feature distribution shift visualiser · alerts · retrain trigger game

Planned
W4
🌐 "Wrap It as an API" — Model Deployment
FastAPI endpoint · send test requests · latency vs accuracy trade-off · A/B test two model versions
Real example: package the Phase 3 churn model as a REST API. CloudSync's backend calls it on every user login
Planned
W5
🏁 Final Capstone — CloudSync Churn Predictor End-to-End
Raw data → feature engineering → model selection → hyperparameter tuning → deployment → monitoring dashboard
